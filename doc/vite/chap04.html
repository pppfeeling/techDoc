Vite + TypeScript 개발 가이드

📅 2025년 5월

🎯 왜 Vite + TypeScript인가?

Vite는 TypeScript를 기본으로 지원하며, 복잡한 설정 없이도 타입 안전성과 빠른 개발 경험을 제공합니다!

🚀 프로젝트 시작하기

TypeScript 템플릿으로 Vite 프로젝트를 생성해보세요:

npm create vite@latest my-react-ts-app -- --template react-ts cd my-react-ts-app npm install npm run dev

🎉 이 명령어로 React + TypeScript + Vite 프로젝트가 바로 생성됩니다!

📁 프로젝트 구조

생성된 프로젝트의 주요 TypeScript 관련 파일들:

my-react-ts-app/ ├── tsconfig.json # TS 설정 ├── tsconfig.node.json # Node용 TS 설정 ├── vite.config.ts # Vite 설정 (TS) ├── src/ │ ├── App.tsx # 메인 컴포넌트 │ ├── main.tsx # 진입점 │ └── vite-env.d.ts # Vite 타입 정의

실시간 타입 체크

📋 권장 tsconfig.json 설정:

{ "compilerOptions": { "target": "ES2020", "useDefineForClassFields": true, "lib": ["ES2020", "DOM", "DOM.Iterable"], "module": "ESNext", "skipLibCheck": true, /* Bundler mode */ "moduleResolution": "bundler", "allowImportingTsExtensions": true, "resolveJsonModule": true, "isolatedModules": true, "noEmit": true, "jsx": "react-jsx", /* Linting */ "strict": true, "noUnusedLocals": true, "noUnusedParameters": true, "noFallthroughCasesInSwitch": true }, "include": ["src"], "references": [{ "path": "./tsconfig.node.json" }] }

🔧 실제 개발 예제

타입 안전한 React 컴포넌트 작성하기:

// src/components/UserCard.tsx interface User { id: number; name: string; email: string; isActive?: boolean; } interface UserCardProps { user: User; onEdit: (id: number) => void; } export const UserCard: React.FC<UserCardProps> = ({ user, onEdit }) => { return ( <div className="user-card"> <h3>{user.name}</h3> <p>{user.email}</p> {user.isActive && ( <span className="active-badge">활성</span> )} <button onClick={() => onEdit(user.id)}> 편집 </button> </div> ); };

🎯 타입 체크 vs 빌드

구분 개발 모드 프로덕션 빌드
타입 체크 IDE에서 실시간 별도 명령어 필요
속도 매우 빠름 ⚡ 보통
에러 처리 즉시 피드백 빌드 시점에 검증
💡 개발 시 타입 체크:
npm run dev # 타입 에러가 있어도 실행됨 tsc --noEmit # 타입만 체크 (빌드 안함)

🔥 Vite의 TypeScript 특별한 기능들

  • 즉시 타입 체크: 파일 저장 시 IDE에서 바로 타입 에러 확인
  • 빠른 Hot Reload: TypeScript 파일 변경 시 즉시 브라우저 업데이트
  • 자동 타입 생성: CSS 모듈, 이미지 등의 타입 자동 생성
  • Path Mapping: tsconfig의 paths 설정 자동 인식

📝 Path Mapping 예제:

// tsconfig.json { "compilerOptions": { "baseUrl": ".", "paths": { "@/*": ["src/*"], "@/components/*": ["src/components/*"], "@/utils/*": ["src/utils/*"] } } } // 사용 예시 import { UserCard } from '@/components/UserCard'; import { formatDate } from '@/utils/dateUtils';

🚨 주의사항 & 팁

  • Vite는 타입 체크를 하지 않고 빌드합니다 - 별도로 tsc --noEmit 실행 필요
  • isolatedModules: true 설정으로 각 파일을 독립적으로 변환
  • ESM import/export만 사용하세요 (CommonJS 사용 금지)
  • vite-env.d.ts 파일로 Vite 관련 타입 정의 제공
