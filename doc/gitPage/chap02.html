챕터 2: 개발 환경에 프로젝트 생성 및 배포

2025.05.29 | GitHub Pages 사용

이제 본격적으로 코딩의 세계로 뛰어들 시간입니다! Vite를 사용해 번개처럼 빠른 개발 환경을 구축하고, 여러분의 React 애플리케이션에 생명을 불어넣어 보겠습니다. 그리고 마법의 주문 gh-pages를 외쳐 GitHub Pages에 여러분의 작품을 선보일 거예요! 🪄

🌱1. Vite로 React + TypeScript 프로젝트 생성

터미널을 열고 다음 명령어를 입력하여 새로운 프로젝트를 생성하세요. 마치 씨앗을 심고 새싹이 돋아나길 기다리는 설레는 순간과 같아요!

npm create vite@latest my-gh-pages-app --template react-ts

여기서 my-gh-pages-app은 여러분의 프로젝트 이름입니다. 자유롭게 변경하세요! --template react-ts 옵션으로 React와 TypeScript가 설정된 프로젝트가 만들어집니다.

프로젝트가 생성되면 해당 폴더로 이동하고 필요한 패키지를 설치합니다.

cd my-gh-pages-app npm install

그리고 React Router를 설치하여 페이지 간 이동 기능을 추가해봅시다.

npm install react-router-dom

이제

npm run dev
명령어로 개발 서버를 실행하여 멋진 시작 페이지를 확인해보세요! (기본 포트: 5173)

🛠️2. GitHub Pages 배포 설정

이제 마법의 가루, gh-pages 패키지를 설치하여 배포 준비를 합니다.

npm install gh-pages --save-dev

다음으로, package.json 파일을 열어 scripts 부분을 수정합니다. 여기에 배포 명령어를 추가할 거예요.

// package.json "scripts": { "dev": "vite", "build": "tsc && vite build", "lint": "eslint . --ext ts,tsx --report-unused-disable-directives --max-warnings 0", "preview": "vite preview", "deploy": "gh-pages -d dist" // ✨ 이 부분을 추가하세요! },

마지막으로 가장 중요한 설정! vite.config.ts 파일에 base 옵션을 추가해야 합니다. GitHub Pages는 프로젝트 페이지를 <username>.github.io/<repository-name>/ 형태로 제공하기 때문에, 우리 앱이 올바른 경로에서 리소스를 찾도록 알려줘야 해요.

// vite.config.ts import { defineConfig } from 'vite' import react from '@vitejs/plugin-react' // https://vitejs.dev/config/ export default defineConfig({ plugins: [react()], base: '/your-repository-name/', // ✨ 이 부분을 추가! 'your-repository-name'을 실제 저장소 이름으로 변경하세요. })

🚨 중요: base 값은 반드시 /저장소이름/ 형식이어야 합니다! (앞뒤 슬래시 포함)
예를 들어 저장소 이름이 my-react-app이라면 /my-react-app/으로 설정합니다.

🌍3. 프로젝트 빌드 및 배포

이제 모든 준비가 끝났습니다! 다음 단계를 따라 프로젝트를 빌드하고 GitHub Pages에 배포해봅시다.

1로컬 변경사항 커밋 (선택 사항)

프로젝트에 변경사항이 있다면 먼저 Git에 커밋합니다.

git add . git commit -m "프로젝트 초기 설정 및 gh-pages 준비" git push origin main # 또는 여러분의 기본 브랜치
⬇️
2프로젝트 빌드

package.json에 설정한 빌드 명령어를 실행하여 프로젝트를 빌드합니다. dist 폴더가 생성될 거예요!

npm run build
dist/ ├── assets/ │ ├── index-xxxxxx.js │ └── index-xxxxxx.css ├── index.html └── ... (기타 정적 파일들)
⬇️
3GitHub Pages에 배포

드디어 마법의 주문을 외칠 시간! package.json에 추가한 deploy 스크립트를 실행합니다.

npm run deploy

이 명령어는 gh-pages 패키지를 사용하여 dist 폴더의 내용을 여러분의 GitHub 저장소의 gh-pages 브랜치에 푸시합니다. (만약 브랜치가 없다면 새로 생성해줍니다!)

⬇️
4배포 확인

배포가 완료되면 GitHub 저장소의 Settings > Pages 탭에서 여러분의 사이트가 게시된 URL을 확인할 수 있습니다. (예: https://<username>.github.io/<repository-name>/)

몇 분 정도 시간이 걸릴 수 있으니, 커피 한 잔의 여유를 가지세요! ☕
