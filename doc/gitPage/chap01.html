챕터 1: GitHub Pages 사용을 위한 기본 설정

2025.05.29 | GitHub Pages 사용

⚙️Github Pages란?

GitHub Pages는 정적 사이트를 호스팅해주는 서비스입니다. 여러분의 React 프로젝트를 빌드하면 정적 파일(HTML, CSS, JS)들이 생성되는데, 이 파일들을 GitHub 저장소에 올려두면 GitHub Pages가 멋지게 웹사이트로 보여줍니다. 마치 여러분의 코드가 온라인에서 살아 숨쉬게 되는 마법과 같죠! ✨

GitHub Pages의 유형:

  • 프로젝트 페이지: 각 프로젝트별로 <username>.github.io/<repository-name> 형태로 생성됩니다.

🛠️준비사항 확인하기

GitHub Pages 배포를 시작하기 전에 다음 사항들이 준비되었는지 확인해주세요:

  • GitHub 계정 (없다면 지금 바로 만드세요!)
  • Git 설치 (버전 관리의 기본이죠!)
  • 배포할 React 프로젝트 (Vite로 만든 프로젝트라면 더욱 좋아요!)

🔧 계정과 Git이 준비되었다면, 이제 저장소를 만들 차례입니다!

📦새로운 GitHub 저장소 만들기

1저장소 생성: GitHub에서 + 버튼을 눌러 'New repository'를 선택하세요.

2이름 설정: 프로젝트와 관련된 멋진 이름을 지어주세요. (예: my-react-gh-pages-app)

3공개 설정: GitHub Pages는 Public 저장소에서 무료로 제공됩니다. (Private 저장소는 GitHub Pro 필요)

4README 추가 (선택): 프로젝트 설명을 위한 README 파일을 추가하면 좋습니다.

5생성 완료: 'Create repository' 버튼을 클릭! 🎉

📜GitHub Pages 설정하기

저장소가 준비되었다면, 이제 GitHub Pages를 활성화할 차례입니다.

1설정 이동: 방금 만든 저장소의 Settings 탭으로 이동하세요.

2Pages 메뉴 선택: 왼쪽 사이드바에서 Pages 메뉴를 클릭합니다.

3Source 선택: 'Build and deployment' 섹션에서 Source를 GitHub Actions로 선택하는 것을 권장합니다. (또는 Deploy from a branch)

만약 Deploy from a branch를 선택한다면:

  • Branch: 배포할 브랜치 (예: main 또는 gh-pages)와 폴더 (예: /(root) 또는 /docs)를 선택합니다. Vite 프로젝트의 빌드 결과물은 보통 dist 폴더에 생성되므로, 이 폴더를 GitHub Pages가 바라보도록 설정해야 합니다.

4저장: 설정을 마치고 Save 버튼을 누르면, 잠시 후 <username>.github.io/ 주소로 사이트가 활성화됩니다!

팁 🔍: 처음에는 GitHub Actions를 사용하는 것이 조금 복잡해 보일 수 있지만, CI/CD 파이프라인을 구축하는 데 매우 유용하며 자동 배포를 쉽게 설정할 수 있습니다. 다음 챕터에서 자세히 다룰게요!
