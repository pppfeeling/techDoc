Sonner (Toast 알림)

📜개요

Sonner는 React 애플리케이션을 위한 세련되고 사용하기 쉬운 토스트(Toast) 알림 라이브러리입니다. 📢 사용자의 현재 작업을 방해하지 않으면서 화면 가장자리에 짧은 피드백 메시지를 일시적으로 표시합니다. Shadcn UI는 이 Sonner 라이브러리를 채택하여 자체 스타일과 함께 Toast 기능을 제공합니다.

Sonner의 주요 특징:

  • 미니멀하고 현대적인 디자인
  • 다양한 토스트 타입 지원 (일반, 성공, 오류, 경고, 정보, 액션, Promise 등)
  • 손쉬운 API: toast() 함수 하나로 대부분의 기능 제어
  • 커스터마이징: 위치, 테마, 지속 시간, 아이콘, 스타일 등 유연한 설정
  • 접근성 고려 (ARIA 속성 자동 관리)
  • 가볍고 성능 우수
{/* JS로 토스트 추가/제거 */}
✨ Sonner를 사용하면 복잡한 설정 없이도 사용자에게 즉각적이고 아름다운 피드백을 제공할 수 있습니다!

⚙️기본 설정 및 설치

먼저 sonner 라이브러리를 설치해야 합니다:


npm install sonner
# 또는
yarn add sonner

Shadcn UI에서 Sonner 기반의 Toast를 사용하려면, Toaster 컴포넌트를 CLI를 통해 프로젝트에 추가합니다. (components/ui/sonner.tsx 파일로 추가됨)

🔧핵심 사용법: <Toaster />와 toast() 함수

1. <Toaster /> 컴포넌트 렌더링

애플리케이션의 루트 레벨 (예: app/layout.tsx 또는 최상위 App 컴포넌트)에 <Toaster /> 컴포넌트를 한 번만 렌더링합니다. 이 컴포넌트가 모든 토스트 메시지가 표시될 컨테이너 역할을 합니다.


// app/layout.tsx 예시
import { Toaster } from "@/components/ui/sonner" // shadcn/ui 경로

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko">
      <body>
        {children}
        <Toaster richColors position="top-center" duration={3000} />
      </body>
    </html>
  )
}

<Toaster /> 컴포넌트의 주요 Props는 이전 Toast 챕터에서 설명한 내용과 동일합니다 (position, richColors, theme, closeButton, duration 등).

2. toast() 함수로 토스트 띄우기

실제 토스트 메시지는 sonner 라이브러리에서 export하는 toast 함수를 호출하여 띄웁니다. 이 함수는 다양한 방식으로 호출될 수 있습니다:

  • 기본 토스트: toast('메시지 내용')
  • 타입별 토스트 (richColors 활성화 시 유용):
    • toast.success('성공 메시지')
    • toast.error('오류 메시지')
    • toast.warning('경고 메시지')
    • toast.info('정보 메시지')
  • 설명(description) 추가: toast('제목', { description: '부가 설명' })
  • 액션 버튼 추가: toast('알림', { action: { label: '확인', onClick: () => console.log('확인됨') } })
  • Promise 처리: toast.promise(myPromiseFn, { loading: '처리 중...', success: '완료!', error: '실패!' })
  • 커스텀 React 컴포넌트: toast(<MyCustomToastComponent />)

toast() 함수의 두 번째 인자로 전달하는 객체를 통해 개별 토스트의 동작과 모양을 상세하게 커스터마이징할 수 있습니다 (duration, icon, className, style, onDismiss, onAutoClose 등).

🎨주요 커스터마이징 옵션

<Toaster /> 전역 옵션:

  • position: 토스트 그룹의 화면 위치 (예: "top-right", "bottom-center").
  • richColors: true로 설정 시 toast.success, .error 등에 맞는 기본 아이콘과 색상 적용.
  • theme: "light", "dark", "system" 중 선택.
  • closeButton: 모든 토스트에 닫기 버튼 표시 여부.
  • duration: 기본 토스트 지속 시간 (ms).
  • visibleToasts: 동시에 보이는 최대 토스트 개수.
  • expand: 토스트가 여러 개일 때 너비 확장 여부 (수직 정렬 시).
  • offset: 화면 가장자리로부터의 간격.
  • toastOptions: 모든 토스트에 적용될 기본 스타일링 옵션 (classNames, style 등). 
    
// 예시: 모든 토스트에 특정 클래스 적용
<Toaster toastOptions={{
  classNames: {
    toast: 'my-global-toast-class',
    title: 'my-title-class',
    description: 'my-description-class',
    actionButton: 'my-action-class',
    cancelButton: 'my-cancel-class',
    closeButton: 'my-close-class',
    // ... 기타 요소 클래스
  },
}} />

개별 toast() 호출 시 옵션:

  • id: 토스트에 고유 ID 부여 (업데이트 또는 수동 제거 시 사용).
  • description: 제목 아래 추가 설명.
  • duration: 해당 토스트만 다른 지속 시간 설정.
  • icon: 커스텀 아이콘 (ReactNode).
  • action: { label: string, onClick: () => void } 형태의 액션 버튼.
  • cancel: { label: string, onClick: () => void } 형태의 취소 버튼.
  • unstyled: true로 설정 시 Sonner의 기본 스타일을 모두 제거하고 직접 스타일링.
  • className, style: 해당 토스트에만 적용할 인라인 스타일 또는 클래스.
  • onDismiss: 토스트가 사라질 때(사용자 클릭 또는 자동) 호출되는 함수.
  • onAutoClose: 토스트가 자동으로 사라질 때 호출되는 함수.

💡사용 예제 (핵심 코드)

이전 Toast 챕터의 예제 코드가 Sonner를 활용한 좋은 예시입니다. 여기서는 몇 가지 추가적인 활용법을 보여드리겠습니다.


"use client"

import { toast } from "sonner"
import { Button } from "@/components/ui/button"
import { useEffect, useState } from "react"

export function AdvancedSonnerDemo() {
  const [toastId, setToastId] = useState<string | number | undefined>();

  // 로딩 후 성공/실패 업데이트 예시
  const handleLoadingUpdate = () => {
    const id = toast.loading("파일을 업로드 중입니다...");
    setToastId(id);

    setTimeout(() => {
      // 성공 시나리오
      toast.success("업로드 완료!", { id, description: "파일이 성공적으로 서버에 저장되었습니다." });
      // 실패 시나리오 (주석 처리)
      // toast.error("업로드 실패", { id, description: "네트워크 오류가 발생했습니다." });
    }, 2500);
  };

  // 커스텀 컴포넌트 토스트
  const CustomToast = ({ name }: { name: string }) => (
    <div className="flex items-center gap-2">
      <span className="text-2xl">🎉</span>
      <div>
        <p className="font-semibold">환영합니다, {name}님!</p>
        <p className="text-sm text-muted-foreground">오늘 하루도 즐겁게 보내세요.</p>
      </div>
    </div>
  );

  return (
    <div className="space-x-2">
      <Button variant="outline" onClick={handleLoadingUpdate}>
        로딩 & 업데이트 토스트
      </Button>
      <Button variant="outline" onClick={() => toast(<CustomToast name="개발자" />)}>
        커스텀 토스트
      </Button>
      <Button variant="outline" onClick={() => toast.message("새로운 메시지", {
        description: "중요한 알림이 도착했습니다. 지금 확인하시겠습니까?",
        action: { label: "지금 확인", onClick: () => alert("확인 페이지로 이동!") },
        cancel: { label: "나중에", onClick: () => console.log("나중에 보기")},
        duration: Infinity, // 사용자가 닫을 때까지 유지
      })}>
        액션 & 취소 버튼 토스트
      </Button>
       {toastId && (
        <Button variant="ghost" size="sm" onClick={() => toast.dismiss(toastId)}>
            (ID: {String(toastId).substring(0,5)}...) 토스트 닫기
        </Button>
       )}
    </div>
  )
}
💡 팁: toast.loading()으로 로딩 토스트를 띄우고, 작업 완료 후 해당 토스트의 ID를 사용하여 toast.success()나 toast.error()로 업데이트할 수 있습니다. 이렇게 하면 사용자에게 일관된 피드백 흐름을 제공할 수 있습니다. duration: Infinity를 사용하면 토스트가 자동으로 사라지지 않고 사용자가 직접 닫거나 액션을 취해야 합니다.
