react-hook-form과 shadcn UI 연동

📜개요

react-hook-form은 React에서 폼 상태 관리, 유효성 검사(validation), 폼 제출 처리를 위한 성능 좋고 간편한 라이브러리입니다. 🎣 Shadcn UI 컴포넌트들은 대부분 제어 컴포넌트(controlled components)로 동작하므로, react-hook-form과 함께 사용하면 강력하고 효율적인 폼을 쉽게 구축할 수 있습니다.

주요 연동 포인트:

  • useForm 훅을 사용한 폼 상태 초기화 및 관리
  • Controller 컴포넌트 또는 register 함수를 사용한 shadcn UI 컴포넌트 연동
  • 유효성 검사 스키마 (예: Zod) 연동
  • 에러 메시지 표시 및 폼 제출 처리

Shadcn UI 공식 문서에서도 react-hook-form과 Zod를 사용한 폼 예제를 제공하고 있으며, 이는 매우 권장되는 조합입니다.

✨ react-hook-form의 Controller를 사용하면 shadcn UI의 제어 컴포넌트들을 쉽게 통합하고, 유효성 검사까지 깔끔하게 처리할 수 있어요!

⚙️기본 설정 및 설치

먼저 필요한 라이브러리들을 설치해야 합니다:


npm install react-hook-form zod @hookform/resolvers
# 또는
yarn add react-hook-form zod @hookform/resolvers
  • react-hook-form: 핵심 라이브러리
  • zod: 스키마 기반 유효성 검사 라이브러리 (선택 사항이지만 강력 추천)
  • @hookform/resolvers: zod와 같은 외부 유효성 검사 라이브러리를 react-hook-form과 연결해주는 리졸버

🔗연동 단계 및 핵심 개념

1. 폼 스키마 정의 (Zod 사용 시)

zod를 사용하여 폼 데이터의 타입과 유효성 규칙을 정의합니다. 이는 TypeScript와 함께 사용할 때 타입 안정성을 크게 높여줍니다.


import * as z from "zod"

const profileFormSchema = z.object({
  username: z.string().min(2, { message: "사용자명은 2글자 이상이어야 합니다." }),
  email: z.string().email({ message: "올바른 이메일 주소를 입력해주세요." }),
  bio: z.string().max(160).optional(),
  notifications: z.boolean().default(false),
  framework: z.enum(["nextjs", "sveltekit", "astro", "nuxt"]),
})

type ProfileFormValues = z.infer<typeof profileFormSchema>

2. useForm 훅 사용

useForm 훅을 사용하여 폼 인스턴스를 생성합니다. 이때 resolver 옵션으로 zodResolver를 전달하여 스키마 기반 유효성 검사를 활성화할 수 있습니다.


import { useForm } from "react-hook-form"
import { zodResolver } from "@hookform/resolvers/zod"

// ... 스키마 정의 ...

function MyProfileForm() {
  const form = useForm<ProfileFormValues>({
    resolver: zodResolver(profileFormSchema),
    defaultValues: { // 폼 필드의 기본값 설정
      username: "",
      email: "",
      notifications: false,
      framework: "nextjs",
    },
    mode: "onChange", // 값 변경 시 유효성 검사 실행 (선택적)
  })

  // ... 폼 제출 핸들러 ...
  function onSubmit(data: ProfileFormValues) {
    console.log("제출된 데이터:", data)
    // API 호출 등 실제 제출 로직
  }
  
  // ... JSX ...
}

3. Shadcn UI 컴포넌트와 Controller 연동

Shadcn UI의 제어 컴포넌트(Input, Checkbox, Select, Switch, RadioGroup 등)는 react-hook-form의 Controller 컴포넌트를 사용하여 연동합니다. Controller는 필드 상태(value, onChange, onBlur)를 shadcn UI 컴포넌트에 전달합니다.

Shadcn UI는 이를 더 쉽게 사용할 수 있도록 <Form>, <FormField>, <FormItem>, <FormLabel>, <FormControl>, <FormDescription>, <FormMessage>와 같은 헬퍼 컴포넌트들을 제공합니다. 이들을 사용하는 것이 권장됩니다.


// ... useForm 및 onSubmit 정의 ...

// Shadcn UI Form 컴포넌트들 import
import { Form, FormControl, FormDescription, FormField, FormItem, FormLabel, FormMessage } from "@/components/ui/form"
import { Input } from "@/components/ui/input"
import { Button } from "@/components/ui/button"
// ... 기타 필요한 shadcn UI 컴포넌트 ...

// JSX 내부
<Form {...form}> {/* useForm에서 반환된 form 객체를 spread */}
  <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-8">
    <FormField
      control={form.control} {/* Controller에 control 전달 */}
      name="username" {/* 스키마에 정의된 필드 이름 */}
      render={({ field }) => (
        <FormItem>
          <FormLabel>사용자명</FormLabel>
          <FormControl>
            <Input placeholder="예: shadcn" {...field} />
          </FormControl>
          <FormDescription>
            공개적으로 표시될 이름입니다.
          </FormDescription>
          <FormMessage /> {/* 유효성 검사 에러 메시지 표시 */}
        </FormItem>
      )}
    />
    
    {/* 다른 필드들 (Email, Checkbox, Select 등)도 유사한 방식으로 추가 */}
    {/* 예: Select */}
    <FormField
      control={form.control}
      name="framework"
      render={({ field }) => (
        <FormItem>
          <FormLabel>선호하는 프레임워크</FormLabel>
          <Select onValueChange={field.onChange} defaultValue={field.value}>
            <FormControl>
              <SelectTrigger>
                <SelectValue placeholder="프레임워크 선택" />
              </SelectTrigger>
            </FormControl>
            <SelectContent>
              <SelectItem value="nextjs">Next.js</SelectItem>
              <SelectItem value="sveltekit">SvelteKit</SelectItem>
              <SelectItem value="astro">Astro</SelectItem>
              <SelectItem value="nuxt">Nuxt.js</SelectItem>
            </SelectContent>
          </Select>
          <FormMessage />
        </FormItem>
      )}
    />
    
    <Button type="submit" disabled={form.formState.isSubmitting}>
      {form.formState.isSubmitting ? "저장 중..." : "프로필 저장"}
    </Button>
  </form>
</Form>

주요 포인트:

  • <Form {...form}>: useForm의 반환값을 Form 컨텍스트 프로바이더에 전달합니다.
  • <FormField control={form.control} name="fieldName" render={({ field }) => (...)} />: 각 폼 필드를 감쌉니다.
    • control: form.control을 전달하여 필드를 react-hook-form에 등록합니다.
    • name: 스키마에 정의된 필드 이름과 일치해야 합니다.
    • render 함수의 field 객체에는 value, onChange, onBlur, ref, name 등이 포함되어 있어 shadcn UI 컴포넌트에 바인딩합니다. (예: <Input {...field} />)
  • <FormItem>, <FormLabel>, <FormControl>, <FormDescription>, <FormMessage>: 필드 레이아웃, 레이블, 입력 컨트롤, 설명, 에러 메시지를 구조화하고 접근성(ARIA 속성)을 자동으로 처리해줍니다.
  • Select, Checkbox, RadioGroup, Switch 등과 같이 value/onValueChange 또는 checked/onCheckedChange prop을 사용하는 컴포넌트들은 field.onChange를 해당 prop에 연결하고, field.value (또는 field.value를 기반으로 한 checked 상태)를 사용합니다.

4. 폼 제출 및 상태 관리

<form> 태그의 onSubmit에는 form.handleSubmit(yourSubmitFunction)을 전달합니다. form.handleSubmit은 유효성 검사를 통과한 경우에만 yourSubmitFunction을 호출합니다. form.formState (예: form.formState.errors, form.formState.isSubmitting, form.formState.isValid)를 통해 폼의 다양한 상태를 확인할 수 있습니다.

💡전체 예제 코드 (간략화)

위에서 설명한 내용을 종합한 간략한 프로필 폼 예제입니다.


"use client"

import { zodResolver } from "@hookform/resolvers/zod"
import { useForm } from "react-hook-form"
import * as z from "zod"

import { Button } from "@/components/ui/button"
import { Checkbox } from "@/components/ui/checkbox"
import {
  Form,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from "@/components/ui/form"
import { Input } from "@/components/ui/input"
import { Select, SelectContent, SelectItem, SelectTrigger, SelectValue } from "@/components/ui/select"
import { toast } from "sonner" // Toaster가 설정되어 있다고 가정

const profileFormSchema = z.object({
  username: z.string().min(2, { message: "2글자 이상 입력해주세요." }),
  email: z.string().email({ message: "올바른 이메일을 입력해주세요." }),
  framework: z.enum(["nextjs", "sveltekit", "astro", "nuxt"], {
    required_error: "선호하는 프레임워크를 선택해주세요.",
  }),
  marketing_emails: z.boolean().default(false).optional(),
})

type ProfileFormValues = z.infer<typeof profileFormSchema>

const defaultValues: Partial<ProfileFormValues> = {
  username: "",
  email: "",
  marketing_emails: true,
}

export function ProfileForm() {
  const form = useForm<ProfileFormValues>({
    resolver: zodResolver(profileFormSchema),
    defaultValues,
    mode: "onChange",
  })

  function onSubmit(data: ProfileFormValues) {
    toast.success("프로필이 성공적으로 업데이트되었습니다!", {
      description: (
        <pre className="mt-2 w-[340px] rounded-md bg-slate-950 p-4">
          <code className="text-white">{JSON.stringify(data, null, 2)}</code>
        </pre>
      ),
    })
    console.log(data)
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-8 w-full max-w-md">
        <FormField
          control={form.control}
          name="username"
          render={({ field }) => (
            <FormItem>
              <FormLabel>사용자명</FormLabel>
              <FormControl>
                <Input placeholder="Your username" {...field} />
              </FormControl>
              <FormDescription>공개적으로 보여질 이름입니다.</FormDescription>
              <FormMessage />
            </FormItem>
          )}
        />
        <FormField
          control={form.control}
          name="email"
          render={({ field }) => (
            // ... 이메일 필드 (위 사용자명 필드와 유사하게 구성) ...
            <FormItem>
              <FormLabel>이메일</FormLabel>
              <FormControl>
                <Input type="email" placeholder="your@email.com" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <FormField
          control={form.control}
          name="framework"
          render={({ field }) => (
            <FormItem>
              <FormLabel>선호 프레임워크</FormLabel>
              <Select onValueChange={field.onChange} defaultValue={field.value}>
                <FormControl>
                  <SelectTrigger>
                    <SelectValue placeholder="프레임워크를 선택하세요" />
                  </SelectTrigger>
                </FormControl>
                <SelectContent>
                  <SelectItem value="nextjs">Next.js</SelectItem>
                  <SelectItem value="sveltekit">SvelteKit</SelectItem>
                  <SelectItem value="astro">Astro</SelectItem>
                  <SelectItem value="nuxt">Nuxt.js</SelectItem>
                </SelectContent>
              </Select>
              <FormDescription>가장 자주 사용하는 프레임워크를 선택해주세요.</FormDescription>
              <FormMessage />
            </FormItem>
          )}
        />
        <FormField
          control={form.control}
          name="marketing_emails"
          render={({ field }) => (
            <FormItem className="flex flex-row items-start space-x-3 space-y-0 rounded-md border p-4">
              <FormControl>
                <Checkbox
                  checked={field.value}
                  onCheckedChange={field.onChange}
                />
              </FormControl>
              <div className="space-y-1 leading-none">
                <FormLabel>마케팅 이메일 수신 동의</FormLabel>
                <FormDescription>
                  새로운 제품, 기능 업데이트, 특별 할인에 대한 정보를 받습니다.
                </FormDescription>
              </div>
            </FormItem>
          )}
        />
        <Button type="submit">프로필 업데이트</Button>
      </form>
    </Form>
  )
}

export default ProfileForm;
💡 핵심 정리: react-hook-form의 useForm으로 폼 상태를 초기화하고, shadcn UI의 <Form>과 <FormField>를 사용하여 각 필드를 연결합니다. field 객체를 shadcn UI 컴포넌트의 props에 적절히 바인딩하는 것이 중요합니다 (예: <Input {...field} /> 또는 <Checkbox checked={field.value} onCheckedChange={field.onChange} />).
