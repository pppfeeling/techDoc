Accordion (아코디언)

📜개요

아코디언은 여러 섹션의 콘텐츠를 수직으로 쌓아놓고, 사용자가 선택한 섹션만 확장하여 보여주는 UI 컴포넌트입니다. 마치 악기 아코디언처럼 접혔다 펴졌다 한다고 해서 붙여진 이름이죠! 🎹

주로 다음과 같은 경우에 유용하게 사용됩니다:

  • FAQ (자주 묻는 질문) 목록
  • 단계별 가이드 또는 튜토리얼
  • 많은 정보를 계층적으로 정리하여 보여줄 때
  • 화면 공간을 효율적으로 사용하고자 할 때
✨ Shadcn UI의 Accordion은 Radix UI의 Accordion Primitive를 기반으로 만들어져 접근성과 키보드 네비게이션이 훌륭해요!

🔧Accordion (메인 컴포넌트) Props

<Accordion> 컴포넌트는 전체 아코디언 컨테이너 역할을 합니다. 다음과 같은 주요 Prop들을 가집니다:

Prop 이름 타입 기본값 설명
type "single" | "multiple" - 아코디언 타입. "single"은 하나만 열리고, "multiple"은 여러 개를 동시에 열 수 있습니다.
collapsible boolean false type="single"일 때, 모든 아이템을 닫을 수 있게 할지 여부입니다. 기본적으로는 하나는 항상 열려있어요.
defaultValue string | string[] - 컴포넌트가 처음 마운트될 때 기본적으로 열려있을 아이템의 value. type="multiple"이면 배열을 사용합니다. (비제어 컴포넌트용)
value string | string[] - 현재 열려있는 아이템의 value. (제어 컴포넌트용)
onValueChange (value: string | string[]) => void - 열려있는 아이템이 변경될 때 호출되는 콜백 함수입니다. (제어 컴포넌트용)
disabled boolean false 아코디언 전체를 비활성화할지 여부입니다. 비활성화되면 아이템을 열거나 닫을 수 없어요. 🚫
orientation "vertical" | "horizontal" "vertical" 아코디언 방향. 현재 shadcn/ui는 주로 수직 방향을 지원합니다.
dir "ltr" | "rtl" - 텍스트 방향 (Left-to-Right 또는 Right-to-Left). 국제화 지원에 유용해요. 🌍
asChild boolean false 기본 렌더링 요소(<div>) 대신 자식으로 전달된 컴포넌트를 렌더링합니다. 커스텀 스타일링이나 특정 HTML 구조가 필요할 때 사용해요.

🧩서브 컴포넌트들

아코디언은 여러 서브 컴포넌트들의 조합으로 구성됩니다. 각자의 역할이 명확해서 사용하기 쉬워요!

Accordion ➡️ AccordionItem ➡️ AccordionTrigger & AccordionContent

🧱AccordionItem

설명: 아코디언 내의 개별 아이템(섹션)을 나타냅니다. 각 아이템은 고유한 value를 가져야 하며, AccordionTrigger와 AccordionContent를 자식으로 가집니다.

부모 컴포넌트와의 관계: <Accordion>의 직접적인 자식으로 사용됩니다.

사용법: <AccordionItem value="unique-item-value">...</AccordionItem>

주요 Props:

Prop타입설명
valuestring아이템을 식별하는 필수 고유 값입니다.
disabledboolean해당 아이템만 비활성화합니다.
asChildboolean기본 <div> 대신 자식으로 렌더링합니다.

👆AccordionTrigger

설명: 각 AccordionItem의 헤더 부분입니다. 클릭하면 연결된 AccordionContent를 열거나 닫는 역할을 합니다. 보통 버튼으로 렌더링됩니다.

부모 컴포넌트와의 관계: <AccordionItem> 내부에 위치하며, 해당 아이템의 콘텐츠 표시를 제어합니다.

사용법: <AccordionTrigger>섹션 제목</AccordionTrigger>

주요 Props:

Prop타입설명
asChildboolean기본 <button> 대신 자식으로 렌더링합니다.

📄AccordionContent

설명: 각 AccordionItem의 실제 내용이 들어가는 부분입니다. AccordionTrigger에 의해 표시되거나 숨겨집니다.

부모 컴포넌트와의 관계: <AccordionItem> 내부에 위치하며, AccordionTrigger와 연결됩니다.

사용법: <AccordionContent><p>여기에 내용이 들어갑니다...</p></AccordionContent>

주요 Props:

Prop타입설명
asChildboolean기본 <div> 대신 자식으로 렌더링합니다.

💡사용 예제 (핵심 코드)

다음은 간단한 아코디언 사용 예시입니다. 실제 프로젝트에서는 @/components/ui/accordion 경로에서 가져와 사용하게 됩니다.


import {
  Accordion,
  AccordionContent,
  AccordionItem,
  AccordionTrigger,
} from "@/components/ui/accordion"

function MyCoolAccordion() {
  return (
    <Accordion type="single" collapsible className="w-full">
      <AccordionItem value="item-1">
        <AccordionTrigger>이것은 첫 번째 질문인가요? 🤔</AccordionTrigger>
        <AccordionContent>
          네, 맞습니다! 첫 번째 아이템의 내용이 여기에 펼쳐집니다. 🌟
        </AccordionContent>
      </AccordionItem>
      <AccordionItem value="item-2">
        <AccordionTrigger>두 번째 아이템도 멋진가요? ✨</AccordionTrigger>
        <AccordionContent>
          물론이죠! 두 번째 아이템은 더욱 흥미로운 내용을 담고 있을지도 몰라요. 🚀
          <ul>
            <li>중요한 포인트 1</li>
            <li>또 다른 중요한 포인트 2</li>
          </ul>
        </AccordionContent>
      </AccordionItem>
      <AccordionItem value="item-3" disabled>
        <AccordionTrigger>이 아이템은 비활성화 되었나요? 🚫</AccordionTrigger>
        <AccordionContent>
          이 내용은 비활성화되어 보이지 않아야 합니다.
        </AccordionContent>
      </AccordionItem>
    </Accordion>
  )
}

export default MyCoolAccordion;
💡 팁: AccordionTrigger 내부에 아이콘 (예: 펼침/닫힘 표시)을 추가하여 사용자 경험을 향상시킬 수 있습니다. shadcn UI는 기본적으로 이를 제공해줘요!
